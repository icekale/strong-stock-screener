# iFinD 接入方案

## 目标

把 iFinD 作为强势股工作台的研究增强层，而不是替代 TickFlow 的行情主源。系统继续保持独立选股和自选股管理定位，不回到大盘日报或全市场复盘工具。

核心分工：

- TickFlow：行情主源，负责日 K、分钟线、实时行情、K 线图和盘中量价判断。
- iFinD：研究增强源，负责行业板块、财务估值、公告新闻、风险事件、股本股东、指数板块和宏观行业数据。
- 东财 / AKShare / 现有源：候选池和兜底源，负责近 20 日涨停池、龙虎榜、涨停池、部分新闻和 fallback。

## 不做什么

- 不用 iFinD 替代 TickFlow 的 K 线和盘中行情。
- 不把系统扩展成日报或大盘复盘产品。
- 不让 AI 直接覆盖用户手写的自选股观察计划。
- 不在筛选新股结果里新增 `empty` 状态；空仓纪律仍只用于自选股或持仓风险。

## 数据源分层

| 能力 | 主源 | 备用 / 补充 | 用途 |
| --- | --- | --- | --- |
| 日 K / 分钟线 / 实时行情 | TickFlow | 现有 K 线 fallback | 量价、均线、K 线图、盘中判断 |
| 20 日内涨停候选池 | 东财 / AKShare | iFinD 可作为后续补充 | 选股入口 |
| 行业 / 概念 / 板块归属 | iFinD | 东财板块 | 结果备注、板块强度 |
| 财务 / 估值 | iFinD | 现有摘要源 | 风险解释、质量标签 |
| 公告 / 新闻 / 风险事件 | iFinD | 东财新闻 | 负面新闻、严重异动、减持、解禁、监管 |
| 指数 / 板块强度 | iFinD | 东财板块行情 | 板块加权评分 |
| 自选股每日观察 | iFinD + TickFlow | 东财补充 | 自动风险提示和观察提醒 |

## 功能增强

### 1. 设置页

新增 iFinD 配置卡：

- API / MCP 是否配置。
- 股票基础资料权限是否可用。
- 公告新闻权限是否可用。
- 指数板块权限是否可用。
- 最近请求耗时。
- 最近错误摘要。
- 当前 fallback 状态。

健康检查必须手动触发，不在页面打开时自动大量请求。

### 2. 首页选股

选股主流程不变：

1. 从近 20 个交易日涨停池生成候选。
2. 用 TickFlow 日 K 和量价规则评分。
3. 用板块强度、风险事件和解释信息增强排序。

iFinD 增强字段：

- 所属行业。
- 所属概念 / 题材。
- 最新公告风险。
- 负面新闻风险。
- 严重异动或监管类风险。
- 估值摘要。
- 财务质量摘要。
- 数据增强状态。

结果页新增“数据可信度”信息：

- 主行情源是否 TickFlow。
- iFinD 增强是否成功。
- 是否发生 fallback。
- 缺失字段数量。
- 最近错误摘要。

### 3. 个股详情页

K 线图继续使用 TickFlow。iFinD 只增强右侧信息区：

- 行业 / 概念 / 板块。
- 财务摘要。
- 估值摘要。
- 最新公告。
- 近期新闻。
- 风险事件。
- 股本股东。

页面重点是提高信息密度，不改变现有工作台视觉风格。

### 4. 自选股观察池

自选股每天可跑一次观察扫描，更新系统字段：

- 今天风险提示。
- 上次复盘结论。
- 所属板块强度。
- 是否有新公告。
- 是否有负面新闻。
- 是否与当日涨停板块有关联。
- 是否触发提醒。

用户手写字段不得被 AI 覆盖：

- 加入理由。
- 计划买点。
- 失效条件。
- 所属题材。

## 评分增强

保留当前强势股短线逻辑，iFinD 不把系统改成长线价值筛选。

建议权重：

- 趋势量价分：60%
- 板块强度分：20%
- 事件解释分：10%
- 基本面 / 估值解释分：10%
- 风险扣分：按严重程度扣 10% 到 30%

风险信号不一定直接剔除，优先做红色标注和扣分。短线情绪票可以保留，但必须说明风险来源。

## 接口抽象

后端新增 `ResearchDataProvider` 级别抽象，避免业务层直接依赖 iFinD：

```text
ResearchDataProvider
├── get_stock_profile(symbol)
├── get_stock_financial_snapshot(symbol)
├── get_stock_valuation(symbol)
├── get_stock_events(symbol)
├── get_stock_news_risk(symbol)
├── get_sector_profile(symbol)
└── get_sector_strength(sector)
```

返回结果统一带来源状态：

```text
source
status
latency_ms
fallback_used
missing_fields
error_summary
updated_at
```

## 实施顺序

### Phase 1：iFinD 配置和健康检查

- 设置页新增 iFinD 配置区域。
- 后端新增 iFinD 健康检查 API。
- 只做轻量探测，不影响现有选股。

### Phase 2：个股详情增强

- 个股详情页接入行业、财务、估值、公告、新闻、风险事件。
- 先只读展示，不参与评分。

### Phase 3：选股结果增强

- 筛选结果补充行业、题材、公告新闻、财务估值和风险事件。
- 记录页保存 iFinD 增强状态，方便追溯。

### Phase 4：自选股每日观察

- 自选股扫描接入 iFinD。
- 写回今天风险提示和上次复盘结论。
- 后续版本再做实时监控和任务流。

## 测试要求

- iFinD 未配置时，现有 TickFlow 选股、K 线、自选股功能不受影响。
- iFinD 请求失败时，页面显示 fallback 状态，不阻塞主流程。
- 健康检查能区分未配置、权限不足、超时、成功。
- 筛选记录能保存 iFinD 增强状态。
- 自选股扫描不得覆盖用户手写观察计划。

## 推荐结论

iFinD 应该做研究大脑，TickFlow 继续做盘中眼睛。这个组合最适合当前产品：TickFlow 判断走势，iFinD 解释原因，东财和现有源提供候选池与兜底。
