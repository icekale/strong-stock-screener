# 股是股非模型体系设计

## 背景

本项目是独立的强势股选股工作台，不是每日复盘项目。本设计把 `/Users/kale/Documents/股是股非拆书/股是股非三部曲拆书与程序化选股规则.md` 中的“三度、量时空、三阳控三阴、均线归位、A/B/C 区、大形态、星线触发”转化为可计算、可展示、可追溯的选股功能。

目标不是替代现有强势股模型，而是同时完成三件事：

1. 增强现有筛选结果，让每只候选股都有股是股非结构评分。
2. 增加独立策略模式，可选择强势股模型、股是股非模型、综合模型。
3. 增强自选股观察池，每天给出结构触发和风险提醒。

## 成功标准

- 首页筛选可以选择策略模式：`强势股模型`、`股是股非模型`、`综合模型`。
- 每个筛选结果展示股是股非总分、分项评分、区位、结构标签、压力风险、操作解释。
- 筛选记录保存策略、模型版本、各模块评分、候选池快照和排序依据，便于追溯短时间内结果差异。
- 自选股页面展示每只股票的股是股非观察状态，并能按触发类型筛选。
- 空仓规则仍只用于自选股或持仓风险，不作为新股筛选状态。
- TickFlow 仍是 K 线与行情主数据源；iFinD 只做行业、公告、新闻、财务、估值、风险事件增强。

## 策略模式

### 强势股模型

沿用现有模型。核心是 20 日内涨停池、红肥绿瘦、量能、MA5、200 日新高、放量上涨、放量滞涨、板块强度、严重异动和负面新闻风险。

筛选状态保持现有枚举：

- `focus`
- `wait_pullback`
- `reduce_risk`
- `data_incomplete`

不新增 `empty` 到新股筛选状态。

### 股是股非模型

仍从候选池开始，但排序和解释主要依据股是股非结构评分。它更重视：

- 左侧量时空压力是否被消化。
- 三阳控三阴是否成立。
- 均线是否归位、密集、穿越、翘头。
- 当前是否处在 A 区、B 区 A 点、C 区。
- 是否接近颈位、箱体上沿、三角末端。
- 星线是否只是环境内触发，而不是孤立形态。

输出操作解释使用中文业务标签，不改变底层筛选状态：

- `强势候选`
- `观察候选`
- `等触发`
- `回避`

### 综合模型

综合模型是默认推荐策略。排序由现有强势股评分和股是股非结构评分共同决定：

- 强势股分数用于确认速度、趋势和短线强度。
- 股是股非分数用于确认厚度、力度、空间和风险地形。
- 板块强度作为加分项，不覆盖风险项。
- 命中高位巨量、C 区、左侧未消化巨量压力等硬风险时，降低排序并标注原因。

## 后端模型

## 规则模块

新增 `app/gsgf_rules.py`，保持独立，避免把 `rules.py` 继续做大。

### 安全区/量时空

输入最近 220 根以上日 K。

识别：

- 左侧 120/250 日高位巨量长上影。
- 巨量倒灌、巨量独阳、巨量假阳。
- 带量下跳缺口且未封堵。
- 当前价格逼近前高压力，但没有放量消化。
- 近 20 日涨幅过大后再次巨量。

输出：

- `pressure_flags`
- `safety_pressure` 分数
- 高风险时加入 `risk_flags`

### 量形态/厚度

窗口默认 40 日，可保留 20/60 日辅助指标。

计算：

- 红量日比例。
- 红量成交占比。
- 阳量均值 / 阴量均值。
- 阳量聚集与阴量聚集。
- 回调是否缩量。

输出：

- `three_yang_controls_three_yin`
- `neutral`
- `three_yin_controls_three_yang`

### 均线归位/力度

使用 MA5/10/20 和 MA10/30/60 两组。

识别：

- 均线密集：`max(ma_group) / min(ma_group) - 1 < threshold`。
- 均线穿越、翘头、顺畅。
- 收盘价站上均线组。
- 放量上涨推动均线从凌乱转归位。

输出：

- `ma_alignment` 分数
- `A区`
- `B区A点`
- `C区`

### 大形态/空间

第一版做轻量几何识别，不做复杂图像拟合。

识别：

- 箱体：近 40-90 日高低区间稳定，突破上沿或回踩上沿。
- 颈位：接近或突破近 60/120 日前高。
- 三角：高点趋平或降低、低点抬高，波动收敛。
- 资金底：长期横盘 + 红量堆积 + 均线归位。

输出：

- `pattern_tags`
- `pattern_space` 分数
- 触发价：箱体上沿、颈位、三角末端高点、星线平台高点。

### 星线/触发

星线只作为环境内触发，不独立生成买点。

识别：

- 星线平台：近 3-5 日至少 2 根小实体，波动收敛，成交缩量。
- 缩量止跌星线。
- 星线平台后放量突破。
- 高位吊颈、平排星线、黄昏类风险。

星线触发只在以下环境有效：

- A 区。
- B 区 A 点。
- 颈位回踩。
- 箱体上沿附近。
- 三角末端。
- 强势再收集后的缩量回调末端。

## 排序逻辑

### 强势股模型排序

保持当前排序逻辑，只把股是股非分析作为展示增强。

### 股是股非模型排序

排序优先级：

1. 排除或降权硬风险：C 区、高位巨量、左侧未消化压力。
2. `total_score` 降序。
3. A 区、B 区 A 点优先。
4. 三阳控三阴优先。
5. 触发标签优先。
6. 板块强度作为同分加权。

### 综合模型排序

建议公式：

```text
combined_score = strong_score * 0.45 + gsgf_score * 0.45 + sector_bonus * 0.10 - hard_risk_penalty
```

硬风险不直接删除，除非用户打开“排除硬风险”筛选项；默认保留但降权并红色标注。

## 数据流

```text
候选池
  -> TickFlow 日K
  -> 现有强势股规则
  -> 股是股非规则
  -> 策略排序
  -> 风险/新闻/严重异动增强
```

## 错误处理

- K 线不足 220 日：`gsgf.zone = unknown`，分数降级，解释写入“K线不足”。
- TickFlow 失败：沿用现有数据源失败状态，不生成伪评分。
- 行业缺失：题材/板块分项给 0，不影响其他技术分。
- 形态冲突：风险标签优先于机会标签。例如同时命中“星线蓄势”和“高位巨量”，操作解释为回避或等确认。
- C 区只用于风险/回避解释，不把新股筛选状态改成 `empty`。
