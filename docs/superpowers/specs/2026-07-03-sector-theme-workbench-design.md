# 板块题材工作台设计

## 背景

当前 `/sectors` 页面主要展示板块资金流排行、估算柱状图和明细表，适合复盘资金方向，但不适合盘中快速比较题材强弱。用户希望参考短线侠首页的交互模式，在板块页面加入“左侧多选题材、右侧分时曲线、下方成分股表”的工作台能力，并明确最终方向为：**概念/题材优先，行业辅助**。

本设计只复用交互模式和信息密度，不复制第三方页面样式，不依赖爬虫抓取短线侠页面。

## 目标

1. 将 `/sectors` 重构为“题材强度工作台”，首屏服务盘中观察。
2. 默认展示概念/题材视角，行业作为辅助属性、筛选项和数据兜底。
3. 支持“板块强度”和“主力流入”两个模式自由切换。
4. 支持多选 2-5 个题材，在同一张分时图中对比走势。
5. 展示选中题材的成分股表，帮助用户快速进入 K 线页或加入自选。
6. 明确展示数据源口径，区分真实资金流、估算资金流和行业兜底。

## 非目标

1. 第一版不做第三方网页爬虫。
2. 第一版不承诺完全复刻短线侠的板块分类体系。
3. 第一版不做自动交易信号，只提供盘中观察和复盘辅助。
4. 第一版不把行业口径伪装成概念/题材口径。

## 页面结构

`/sectors` 首屏采用两栏工作台布局：

- 顶部标题栏：题材强度工作台、数据源状态、刷新按钮、采样时间。
- 左侧栏：题材/概念多选列表。
- 右侧上方：多题材分时曲线图。
- 右侧中部：关联子题材标签。
- 右侧下方：选中题材成分股表。

### 左侧题材列表

左侧顶部提供模式切换：

- `板块强度`
- `主力流入`

切换模式不会清空已勾选题材，只改变：

- 列表排序指标
- 列表右侧数值
- 右侧分时图单位
- 成分股表中的强调字段

题材列表每项展示：

- 题材名称
- 今日涨停数量
- 当前强度或主力净流入
- 成交额
- 领涨股
- 数据口径标记：概念、行业兜底、估算资金流

列表支持：

- 搜索题材名
- 多选题材
- 最多同时勾选 5 个题材
- 默认选中强度排名前 3 个题材

### 右侧分时曲线

分时图显示已勾选题材的盘中走势。

`板块强度` 模式：

- Y 轴为综合强度分。
- 强度分由涨停数量、平均涨幅、上涨家数占比、成交额、领涨股强度组合计算。
- 曲线用于观察题材是否持续抬升、横盘衰减、快速退潮或午后回流。

`主力流入` 模式：

- 优先使用真实主力净流入数据。
- 如果数据源没有真实主力资金字段，则使用成交额与涨跌幅估算，并在 UI 标注“估算口径”。
- 曲线用于验证强题材是否有资金持续确认。

图表能力：

- 鼠标悬停显示时间、题材名、指标值。
- 图例可点击隐藏单条曲线。
- 横轴为交易时间。
- 非交易时段显示最近一次采样结果。

### 关联子题材

右侧中部展示当前选中题材的关联标签，例如：

- 半导体设备
- 光模块
- 高速铜缆
- 算力租赁

标签来源优先级：

1. 概念映射数据源返回的关联概念。
2. 选中题材成分股的其他概念交集。
3. 行业字段兜底。

### 成分股表

成分股表显示当前选中题材下的个股，字段包括：

- 名称 / 代码
- 行业
- 所属题材
- 涨幅
- 成交额
- 换手率
- 涨停/连板信息
- 竞价涨幅
- 竞价成交额
- 封单或风险提示
- 操作：看 K 线、加入自选

默认排序：

- `板块强度` 模式：涨停/连板优先，其次涨幅、成交额、换手率。
- `主力流入` 模式：主力净流入优先，其次成交额、涨幅、换手率。

点击股票进入 `/stock/[symbol]`，并带上 `from=sectors`、股票名称、行业和题材上下文，返回时回到 `/sectors`。

## 数据设计

### 数据源职责

TickFlow 继续作为行情主源：

- 实时涨幅
- 成交额
- 换手率
- 个股行情
- 竞价相关字段

题材/概念归属由补充数据源提供：

- TDX / iFinD / 同花顺类数据源提供概念映射。
- 如果概念映射不可用，系统降级为 TickFlow 行业聚合。

资金流口径：

- 优先真实资金流。
- 无真实资金流时使用估算资金流。
- UI 和 API 都必须返回 `flow_status`，不能只返回数字。

### 后端聚合模型

新增一个板块工作台聚合接口：

`GET /api/sectors/workbench`

查询参数：

- `mode=strength|main_flow`
- `scope=theme|industry|auto`
- `limit=20`
- `selected=题材A,题材B`
- `stock_limit=50`

响应包含：

- `scope`
- `mode`
- `trade_date`
- `generated_at`
- `source_status`
- `themes`
- `selected_themes`
- `series`
- `related_tags`
- `stocks`

主题项字段：

- `name`
- `scope`
- `limit_up_count`
- `strength_score`
- `main_flow_cny`
- `turnover_cny`
- `change_pct`
- `leader`
- `member_count`
- `source`
- `flow_status`

分时序列字段：

- `name`
- `scope`
- `metric`
- `points`

每个点：

- `time`
- `value`
- `sampled_at`

成分股字段：

- `symbol`
- `name`
- `industry`
- `themes`
- `pct_change`

- `limit_up`

### 采样与缓存

为支持分时曲线，后端需要保存盘中采样快照。

第一版采样策略：

- 页面刷新或调用接口时，若处于交易时段，追加一条当前快照。
- 后台任务流可在后续版本中每 1-3 分钟自动采样。
- 每个交易日按 `data/sectors/YYYY-MM-DD.json` 保存轻量样本。
- 每个题材每个采样点只保存聚合指标，不保存完整 K 线或全部原始行情。

缓存控制：

- 实时接口缓存 30-60 秒。
- 题材映射缓存按交易日复用。
- 采样文件只保留最近一段时间，避免无限增长；默认保留 60 个交易日。

## 降级与错误处理

1. 概念映射不可用：
   - 切换为行业聚合。
   - 页面显示“当前为行业兜底”。

2. 真实主力资金不可用：
   - 使用估算资金流。
   - 页面显示“主力流入口径：估算”。

3. TickFlow 行情不可用：
   - 页面显示错误卡片和最近一次缓存数据。
   - 不用旧数据伪装实时。

4. 某题材成分股为空：
   - 显示空状态：“该题材暂无可用成分股，可能是映射源缺失或非交易时段数据为空。”

## 前端实现边界

新增或改造组件：

- `SectorPageWorkspace`
- `SectorThemeWorkbench`
- `SectorThemeSelector`
- `SectorModeTabs`
- `SectorIntradayChart`
- `SectorRelatedTags`
- `SectorStockTable`

样式遵循现有工作台：

- 保持 Ant Design + Tailwind 的当前视觉系统。
- 不使用营销式大卡片。
- 信息密度偏高，但表格和按钮必须对齐。
- 上涨红色、下跌使用当前统一低饱和绿色。

## 测试计划

后端测试：

1. 概念映射可用时，`/api/sectors/workbench` 返回 `scope=theme`。
2. 概念映射不可用时，自动降级到 `scope=industry` 并返回状态说明。
3. `strength` 模式按强度分排序。
4. `main_flow` 模式按资金流排序。
5. 分时采样只保存聚合快照，不保存完整原始行情。
6. 缓存保留策略不会无限增长。

前端测试：

1. `/sectors` 显示双模式切换。
2. 切换 `板块强度` / `主力流入` 不清空已选题材。
3. 成分股表包含行业、题材、涨幅、成交额、竞价字段。
4. 点击股票进入 K 线页时带 `from=sectors` 上下文。
5. 概念数据缺失时显示行业兜底提示。

## 实施顺序

1. 后端先提供 `/api/sectors/workbench` 的行业兜底版本，保证 TickFlow 实时行情可用。
2. 接入概念/题材映射，把默认视图切到 `scope=theme`。
3. 前端替换 `/sectors` 首屏为题材工作台。
4. 增加分时采样文件，支持曲线逐步积累。
5. 完善主力流入真实/估算口径标注。
6. 加入股票跳转和加入自选操作。

## 验收标准

1. 用户打开 `/sectors`，默认看到概念/题材工作台，而不是旧资金流排行页。
2. 可以在左侧勾选多个题材，并在右侧看到多条曲线。
3. 可以在“板块强度”和“主力流入”之间切换。
4. 可以看到选中题材的成分股表，并进入个股 K 线页。
5. 页面明确显示数据源与口径：TickFlow 行情、概念映射来源、真实资金流或估算资金流。
6. 数据源缺失时页面降级清晰，不出现空白或误导性数字。
