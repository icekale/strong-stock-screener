# 市场总览工作台重构设计

**状态：已确认，待实施**

本设计取代 `2026-07-10-app-shell-workbench-design.md` 相关的未合并 UI 方案。实现从当前 `main` 的业务能力重新组织界面，不合并或复用 `codex/app-shell-workbench` 分支的视觉和组件资产。

## 目标

- 将产品默认入口改为面向交易决策的市场总览，而不是完整选股配置页。
- 保留强势选股和早盘竞价的既有数据、规则、模型、异步任务和个股详情能力。
- 减少并列入口、重复说明和技术性标签，让候选、风险与市场确认信号优先可见。
- 保持桌面端高密度操作效率；移动端支持浏览候选、风险和个股，不承载复杂筛选或模型维护。

## 范围与非目标

本次重构只修改 `apps/web` 的导航、路由组织、视觉 token、布局和展示状态。后端接口、选股规则、竞价 Top3 训练与运行逻辑、TickFlow 接入、数据源选择均不修改。

不新增仪表盘专用聚合 API，不在首页触发选股、竞价 Top3 或训练的重算任务，不改变任何交易信号的语义。首页只能读取已有结果和快照。

## 产品信息架构

### 市场

| 入口 | 路径 | 用途 |
| --- | --- | --- |
| 市场总览 | `/` | 默认首页；以决策队列、市场脉冲、板块热度和异动为主。 |
| 强势选股 | `/screener` | 保留完整筛选条件、异步运行、候选结果与加入自选能力。 |
| 竞价雷达 | `/auction` | 保留竞价快照、模型 Top3、实盘确认、回测和复盘能力。 |
| 板块与热图 | `/market` | 将板块雷达与市场热图合并为一个分析页，用页内标签切换。 |

### 观察

| 入口 | 路径 | 用途 |
| --- | --- | --- |
| 自选与风险 | `/watchlist` | 自选池、标签、分组、风控动作和跟踪结果。 |
| 情绪与复盘 | `/sentiment` | 涨停池、连板、短线情绪和日内复盘。 |

### 系统

`/system` 合并模型维护和数据源配置，并使用页内标签区分：模型健康、训练与维护、数据源和缓存状态。低频系统功能不再占据一级导航。

`/sectors`、`/heatmap`、`/model-maintenance` 和 `/settings` 保留为兼容跳转，分别导向对应的新页或标签。股票详情路径保持不变。

## 默认首页

首页采用市场总览布局，按照操作优先级排列：

1. **优先处理区**：竞价 Top3、强势选股候选、自选风险。每行均可直接进入个股详情或原有执行页。
2. **市场确认区**：按交易时段变化的市场脉冲、情绪或成交结构图表。
3. **结构分析区**：板块热度预览和市场异动流，均提供进入完整分析页的单一入口。

页面根据当前交易时段切换标题和市场确认区，始终保留手动切换。盘前强调竞价候选，盘中强调选股确认和板块强度，收盘强调情绪和复盘；优先处理区的排序规则不因时段变化而改变。

## 视觉和交互原则

- 使用冷灰画布、白色工作面、深靛导航和蓝色主操作。上涨、下跌、风险只使用红、绿、橙等市场语义色。
- 使用系统中文字体；页面标题 22px、区域标题 15px、正文 13px、元信息 11px。删除双语眉题、反复说明和常驻技术提示。
- 工作面不采用页面级浮卡或嵌套卡片。只有独立决策单元、表格和工具面板使用不超过 8px 的圆角和细边界。
- 工具动作使用图标或图标加文字；数据刷新、运行筛选等明确命令保留文字按钮。数字列右对齐，表格列头固定，标的行可通过明显 hover 和焦点状态进入详情。
- 桌面端在 980px 以上展示分组侧栏；窄屏用抽屉导航，首页只保留决策队列、风险和个股入口。复杂筛选与系统维护在窄屏提示使用桌面端完成。

## 组件边界

- `ProductShell`：顶部状态、桌面分组导航、移动抽屉和当前路由高亮；不读取业务数据。
- `PageCommandBar`：紧凑标题、时间/数据状态和页面主操作；不承载描述性文案。
- `DecisionQueue`：只渲染 Top3、最新选股结果和自选风险的统一列表，并处理独立加载、空和错误状态。
- `MarketPulse`、`SectorHeatmapPreview`、`MarketFeed`：独立读取结果并提供到完整页面的链接，不共享失败状态。
- `DataState`：统一骨架、空结果、陈旧快照和可恢复错误的展示。技术错误仅在用户展开后显示原始信息。
- 原有选股、竞价、板块、热图、自选、情绪、模型维护工作区只替换页面框架和相邻展示组件，不重写其业务状态机。

旧的 `AppShell`、`WorkbenchPage` 和旧 token 逐页替换。当所有页面完成迁移后，删除仅由旧布局使用的组件和样式。

## 数据加载与性能

首页在首屏通过 `Promise.allSettled` 并行读取下列已有接口：

- 最新选股结果：`getLatestScreenRun`。
- 市场概览：`getMarketOverview`。
- 竞价 Top3：`getAuctionModelTop3(tradeDate, { cacheOnly: true })`。
- 自选风险：`getWatchlistGsgfStatus`。
- 板块预览：`getSectorRadar` 或热图概览接口。
- 情绪/异动摘要：使用已有摘要接口，不额外触发运行任务。

任一请求失败时，其面板保留最近成功数据（若有）并显示陈旧或错误状态；其他面板继续渲染。顶部刷新只重试可刷新读取，不启动模型重算。Top3 重新生成入口保留在竞价雷达页。

## 迁移顺序

1. 先建立新 token、`ProductShell`、导航结构和可访问的基础状态组件。
2. 将首页替换为市场总览，并把完整选股工作台移动至 `/screener`。
3. 迁移竞价、个股详情、自选和情绪页面的外层框架。
4. 合并板块/热图与模型/数据源的入口，添加兼容跳转。
5. 删除不再使用的旧壳层组件和样式，保留业务组件与路由数据契约。
