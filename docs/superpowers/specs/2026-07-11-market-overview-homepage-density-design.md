# 市场总览首页信息密度优化设计

**状态：已确认，修订待实施**

本设计在现有市场总览工作台基础上调整首页的信息架构和视觉层级。它不修改后端接口、选股规则、竞价模型、行情口径或缓存策略。

## 目标

- 让首页在三秒内回答两个问题：资金流向哪里，当前市场是否支持交易。
- 将板块资金流放在首屏最上方，并从文字榜单改成可比较方向和量级的图形。
- 提升成交额、涨跌家数、涨跌停和情绪分等关键数据的辨识度。
- 删除重复信息，降低纵向长度，提高桌面和移动端的信息密度。
- 移除首页决策队列，使首页只承担市场方向判断，不重复竞价、选股和自选页面的执行工作流。

## 非目标

- 不新增或修改市场、板块、情绪、竞价和选股接口。
- 不在首页触发模型运行、筛选任务或行情重算。
- 不改变净流额、市场广度、涨跌停和情绪指标的业务语义。
- 不重构应用外壳、导航或其他工作台页面。

## 页面结构

首页按以下顺序渲染：

1. **市场方向首屏**：左侧板块资金流，右侧市场状态。
2. **指数窄栏**：展示上证、深证、创业板和科创 50 的点位与涨跌幅。

首页在指数窄栏后结束。竞价 Top3、强势选股和自选风险分别保留在竞价雷达、强势选股和自选风险页面，继续通过现有侧边栏进入。

现有 `MarketFeed` 不再在首页渲染。它展示的板块净流入、净流出和情绪摘要已由首屏覆盖，继续保留会造成重复。

桌面端首屏采用约 `65% / 35%` 的双栏比例。板块资金流是稳定主区，市场状态是紧凑确认区。指数窄栏紧接首屏，不再放在独立大面板内。

## 板块资金流

### 桌面端

使用以零轴为中心的水平资金流图：

- 左侧展示净流出 Top 4，右侧展示净流入 Top 4。
- 条形长度只编码净流额绝对值。
- 流入和流出分别按各自方向的最大绝对值归一化。两侧不共享一个最大值，避免极端流出值使全部流入条不可辨认。
- 流入使用 `--market-rise`，流出使用 `--market-fall`；背景使用相应的低饱和语义色。
- 每个方向使用稳定的三层行结构：第一行展示板块名和净流额，第二行展示金额条，第三行展示领涨股、板块涨幅和强度。
- 板块名和净流额不覆盖在金额条上。文字与条形共享同一个方向容器，但使用正常文档流布局，避免比例条缩放时出现标签与条形错位。
- 板块涨幅、强度和领涨股作为次级信息，不参与条形长度或颜色深浅编码。
- 面板标题保留数据源和直接/估算口径，使用次级文字；右上角保留进入完整板块页的链接。

排序直接沿用接口返回的 `inflow` 和 `outflow` 方向排名，并各截取前四项。净流额为空的项目不参与图形排名；无可用项目时使用现有 `DataState` 空状态。

### 移动端

在窄于桌面双栏断点时，将中轴图改为“净流入 / 净流出”分段切换：

- 默认显示净流入。
- 每个方向展示四项。
- 每项显示板块名、净流额、横向金额条、领涨股、板块涨幅和强度。
- 分段控件有清晰的选中、键盘焦点和点击状态，不使用只靠颜色区分的状态。
- 切换只改变本地展示，不重新请求接口。

## 市场状态

市场状态面板集中展示：

- 总成交额及较昨日变化。
- 上涨家数 / 下跌家数。
- 涨停家数 / 跌停家数。
- 情绪阶段和情绪分。

主要数字使用 `20–26px` 固定字号、`tabular-nums` 和高对比文字。标签和数据口径使用现有次级文字 token。

上涨/下跌增加一条比例条，宽度按有效上涨与下跌家数计算。平盘家数不参与红绿比例，但可保留在辅助说明或可访问标签中。分母为零时比例条保持中性，不产生 `NaN` 样式。

市场数据和情绪数据继续独立处理状态：

- 市场接口成功、情绪接口失败时，成交额和广度照常显示，情绪区域显示可重试错误。
- 情绪成功、市场失败时，情绪数据照常显示，市场区域显示可重试错误。
- 陈旧数据保留最近成功值，并显示现有陈旧状态提示。

## 指数窄栏

指数窄栏展示接口返回的指数列表，当前目标为四项：上证、深证、创业板和科创 50。

- 桌面端单行四列。
- 中等宽度可折为两列。
- 手机端使用两列网格，保证名称、点位和涨跌幅不溢出。
- 点位为主文字，涨跌幅使用市场语义色，不再给每项单独套装饰卡片。

## 首页职责与请求精简

首页不再渲染 `DecisionQueue`，也不再请求竞价 Top3 和最近一次强势选股结果。这样可避免首页为非核心内容等待缓存或模型相关接口，并减少首次加载的并行请求数量。

此调整只改变首页组合方式：

- `DecisionQueue` 组件和既有业务逻辑不删除，避免影响其他潜在使用方。
- 竞价 Top3 继续在 `/auction` 展示和运行。
- 强势选股继续在 `/screener` 展示和运行。
- 自选风险继续在 `/watchlist` 展示。
- 侧边栏已经提供这些入口，首页不再增加重复的快捷导航。

## 组件与数据边界

- `MarketOverviewWorkbench`：只加载市场总览、板块资金流和情绪摘要，保持现有并行加载和 latest-only 防竞态逻辑；不再维护首页竞价和选股状态。
- `SectorHeatmapPreview`：改造成桌面中轴图和移动分段榜单，继续接收现有 `PanelState<SectorRadarResponse>`。
- `MarketPulse`：重组为紧凑市场状态和指数窄栏，继续分别接收市场与情绪状态。
- `DecisionQueue`：组件本身不改动，但不再由首页引用。
- `marketOverview.ts`：增加可测试的资金流截取/归一化和市场广度比例纯函数。
- `globals.css`：只增加本页需要的稳定布局、条形和响应式规则，继续使用现有产品 token。

不引入新的图表库。条形图使用语义化 HTML 和 CSS 宽度实现，减少首屏 JavaScript 体积，并保留屏幕阅读器可读的文本值。

## 可访问性与交互

- 所有涨跌方向同时使用符号、文字和值，不只依赖红绿颜色。
- 图形条目提供包含板块名、方向、净流额、涨幅、强度和领涨股的可访问文本。
- 移动端分段控件使用标准按钮语义、`aria-pressed` 或标签页语义，并有可见焦点。
- 链接和刷新操作继续使用现有焦点样式。
- 不增加装饰动画；切换状态使用即时更新或不超过 150ms 的颜色过渡。

## 测试与验收

### 单元和结构测试

- 资金流各方向只返回前四项，并忽略净流额为空的项目。
- 流入和流出分别归一化，最大项宽度为 100%，其余项按绝对值比例计算。
- 全零或空数据不产生非有限比例。
- 市场广度比例正确处理上涨、下跌、平盘和零分母。
- 首页只渲染板块资金流、市场状态和指数窄栏，不再渲染 `MarketFeed` 或 `DecisionQueue`。
- 首页刷新只调用市场总览、板块资金流和情绪摘要三个接口。
- 桌面资金流标签、金额条和辅助信息使用正常文档流，不存在覆盖金额条的绝对定位标签。

### 视觉验收

- `1440px`：板块资金流与市场状态同处首屏，指数窄栏紧随其后，页面无水平滚动。
- `781px`：双栏可合理收窄或按断点重排，关键数字和板块标签不截断到不可读。
- `390px`：资金流使用分段切换，市场状态重排为两列，指数为两列。
- 所有视口下不存在文本重叠、按钮溢出、动态内容引发布局跳动或空状态占据过大高度。
