# 多周期缠论工作台设计

## 背景

项目当前以 TickFlow 日 K、强势股筛选、竞价 Top3、GSGF 证据和自选股观察池构成短线研究工作台。个股详情页已有 ECharts 主图叠加层，但只能绘制 GSGF 证据和砖形图，不能表达缠论的分型、笔、线段、中枢、买卖点或背驰。

本功能增加一个可复核的多周期缠论分析链路，并将其以三种方式提供：

- 个股详情页的轻量缠论叠加与“进入缠论工作台”入口；
- `/chanlun` 独立工作台，按代码或名称分析任意股票；
- 选股结果中的结构摘要、筛选条件和多周期共振分。

缠论结构计算参考 `czsc` 的公开实现：其仓库 LICENSE 为 Apache-2.0，核心分型、笔和中枢在 1.x 版本由 Rust 原生扩展提供。应用只消费本项目定义的数据模型，不暴露 `czsc` 的内部对象。参考站点展示的是产品功能和视觉语言，不复制其专有实现、页面代码或声称逐信号一致。

## 目标

1. 在日线、60 分钟、30 分钟和 5 分钟级别输出可解释的缠论结构。
2. 在 K 线主图完整呈现细笔、线段、实中枢、虚中枢、买卖点和背驰系数，区分观察、暂定、确认和最终状态。
3. 让强势选股候选和自选股获得多周期结构摘要，不替换既有强势股、竞价 Top3 或 GSGF 的主逻辑。
4. 提供去重预警、历史回放和模拟盘订单草案；订单必须人工确认，只在本地模拟账户执行。
5. 在数据不足、数据源异常或结构库不可用时明确降级，绝不以旧数据或未确认信号生成可执行订单。

## 非目标

- 不接入真实券商、QMT、PTrade、miniQMT 或任何真实资金通道。
- 不自动提交真实订单，不保存券商凭证。
- 不以未收盘 K 线、未来 K 线或事后修正的结构做回测、预警或模拟成交。
- 不改变竞价 Top3 的训练目标、模型特征或现有 GSGF 状态定义。
- 不以全 A 股票池每 5 分钟重算缠论；分钟级计算限定在现有候选池、自选股和活跃订单标的。
- 不承诺与任一商业缠论产品的买卖点、背驰或视觉坐标逐项完全一致。

## 总体架构

```text
TickFlow 当日 1m / 5m / 30m / 60m
             |                         
             v                         
  LocalMinuteBarStore <--- TDX/mootdx 多日分钟历史补数
             |                         
             v                         
  Trading-session aggregation (1m -> 5m / 30m / 60m)
             |
             v
  ChanlunAdapter (czsc + project rule layer)
             |
             +--> Stock K-line overlay and /chanlun workbench
             +--> Screener summary, filters and ranking
             +--> Alert deduplication
             +--> Paper-order drafts, replay and review
```

后端新增独立 `chanlun` 服务域，不修改现有 `StrongStockScreener` 的基础候选生成、竞价模型训练或 GSGF 规则。`czsc` 仅被适配器引用；API、存储和前端只认识本项目 Pydantic/TypeScript 模型。

## 数据源与时间口径

### 实时与历史职责

| 数据 | 首选来源 | 使用方式 | 约束 |
| --- | --- | --- | --- |
| 日 K | TickFlow | 前复权日线，日线环境和历史结构 | 与现有个股详情一致 |
| 当日 1m / 5m / 30m / 60m | TickFlow | 盘中刷新和当日归档 | 已验证可直接获取 5m、30m、60m；当前授权返回最近一个交易日 |
| 多日分钟历史 | `mootdx` 通达信 | 首次查看和回放时补数 | `category` 7/8/10/11 对应 1m/5m/30m/60m；作为可选源，需健康检查 |
| 持久化历史 | 项目本地 SQLite | 分析、回放、回测的权威输入 | 不依赖 Windows free-stockdb |

`free-stockdb` 保持竞价模型的离线训练/回测用途，不进入缠论实时分析路径。

### 分钟归档

- 新增 `data/chanlun/minute.sqlite3`，按 `symbol + timestamp` 唯一保存原始 1 分钟 OHLCV、来源、抓取时间、交易日和复权口径。
- TickFlow 每次刷新仅追加或更新当前交易日尚未闭合的分钟数据；闭合 K 线不可被后续行情静默覆盖，任何修订必须留下来源和修订时间。
- 5m、30m、60m 均由已闭合 1 分钟 K 按 A 股上午 `09:30-11:30`、下午 `13:00-15:00` 分段聚合。午休不跨桶，收盘前不足完整周期的 K 标为未闭合。
- 首次工作台查询需要历史而本地库不足时，后台从 `mootdx` 回填；接口返回 `backfilling`，页面可显示已有数据但不生成交易信号，直至满足周期最小样本数。
- 不同复权口径不可混合分析。若历史补数只有未复权分钟 K，快照标记 `raw_unadjusted`；发生除权缺口且无法取得一致复权因子时，跨除权结构标记为不可用，而不是与 TickFlow 前复权日 K 拼接。

### 时间与确认

- 日线仅在交易日收盘后进入 `confirmed`；日内周期仅在相应 K 收盘后进入 `confirmed`。
- 未收盘 K 线可用于显示 `observing` 结构，但不能进入筛选、预警、回测或订单草案。
- 5m、30m、60m 周期在收盘边界后等待一个短暂源延迟窗口，再拉取、归档并重算。
- 所有分析响应携带 `calculated_at`、`last_closed_bar_at`、`source_status`、`adjustment_mode` 和 `rule_version`。

## 结构计算与规则版本

### 适配器边界

新增 `ChanlunAdapter`：

1. 将项目 `KlineBar`/分钟 bar 转为 `czsc` 所需的标准 bar。
2. 调用固定版本的 `czsc` 生成包含关系处理后的 K 线、分型、笔和中枢。
3. 将结果映射为本项目 `ChanlunAnalysis`，其中每个对象都有时间、价格、方向、确认状态、来源 bar 范围和规则版本。
4. 由项目规则层根据结构对象、MACD 和前置中枢数计算买卖点、虚中枢和背驰；不依赖前端二次推导。

Docker 镜像把 `czsc` 作为显式依赖，并安装其运行时所需的 OpenMP 运行库。镜像构建阶段必须执行 Python import smoke test；应用运行时仍保留 `unavailable` 降级，保证其它页面不被单一指标依赖阻断。

### 状态模型

所有分型、笔、线段、中枢、买卖点和背驰统一使用以下状态：

- `observing`：依赖未收盘 K，只用于图表观察。
- `provisional`：已出现候选分型或候选中枢，但后续一笔尚未完成。
- `confirmed`：所需 bar 已收盘且已满足当前规则确认条件。
- `final`：后续线段确认后仍保留的结构，仅用于历史复盘标注。

选股、预警和模拟订单只接受 `confirmed` 或 `final`。

### 买卖点与背驰

规则层以版本化 `cl-v1` 输出下列标准化标签：

- 一买/一卖、二买/二卖、三买/三卖；
- 盘整一买/盘整一卖、盘整背驰；
- 顶背驰/底背驰；
- 类三买/类三卖、次二买/次二卖、次三买/次三卖；
- 实中枢和虚中枢。

背驰比较已完成的同向走势段：价格创新高/低与 MACD 力度共同成立才生成事件。力度以同向 MACD 柱体绝对面积/长度的比值归一化为背驰系数；响应同时保存当前段、参考段、价格极值、MACD 力度和中枢数量。一个中枢后的背驰分类为盘整背驰，两个及以上中枢后的背驰分类为顶/底背驰。每一项结果都包含 `rule_version`，未来调整口径不会污染历史回放。

虚中枢仅表示当前已知笔的候选重叠区；它不会被当作确认中枢或订单依据。

## 多周期决策规则

首版模拟盘买入草案遵循固定的、可回测的共振规则：

1. 日线不处于确认卖出结构或确认卖方背驰风险状态。
2. 60m 与 30m 至少一个出现确认的买类结构。
3. 5m 出现确认买点，且实时价格没有超过草案的最大允许追价阈值。
4. 所有参与周期的数据状态必须为 `fresh`，且每个信号都为 `confirmed` 或 `final`。

卖出草案使用对应反向条件。规则不构成投资建议；展示内容必须明确区分结构信号、风险条件和人工决策。

周线保留在 K 线浏览中，不进入首版的自动共振条件。

## API 与持久化契约

### 分析与工作台

- `GET /api/chanlun/stocks/{symbol}/analysis`
  - 参数：`period`（`1d`、`60m`、`30m`、`5m`）、`lookback`、`include_observing`。
  - 返回：bars、结构图层、状态摘要、数据源状态、规则版本和可用性。
- `GET /api/chanlun/stocks/{symbol}/workspace`
  - 一次返回日线、60m、30m、5m 的摘要和工作台所需的默认主周期数据。
- `POST /api/chanlun/stocks/{symbol}/backfill`
  - 建立单标的历史补数任务；不在同步 HTTP 请求中做长时间 TDX 拉取。
- `GET /api/chanlun/replays/{symbol}`
  - 按周期、日期范围和规则版本逐 bar 返回历史结构状态与信号事件。

### 选股与预警

- 选股响应新增可选 `chanlun_summary`，包括四周期方向、最近确认信号、最近确认背驰、信号年龄、数据新鲜度和 `confluence_score`。
- 选股过滤新增仅对已计算项生效的缠论条件；没有缠论数据的候选默认保留并标明“结构数据不足”。
- `GET /api/chanlun/alerts` 读取事件；后台刷新只在状态首次进入确认时创建新预警。
- 预警唯一键为 `symbol + period + signal_type + confirmed_at + rule_version`，用于防止重复发送。

### 模拟盘

- `POST /api/chanlun/paper-orders/drafts` 根据已确认多周期信号生成订单草案。
- `POST /api/chanlun/paper-orders/{id}/approve` 必须由用户人工确认，才允许进入模拟挂单。
- `POST /api/chanlun/paper-orders/{id}/cancel` 撤销待确认或挂单草案。
- `GET /api/chanlun/paper-account` 返回现金、持仓、订单、收益和审计记录。

模拟盘数据保存在 `data/chanlun/paper.sqlite3`。订单状态固定为：`draft`、`awaiting_confirmation`、`simulated_open`、`filled`、`rejected`、`expired`、`cancelled`。每次状态变化保留信号快照、规则版本、报价时间、拒绝原因和操作者时间。

## 图表与交互

### `/chanlun` 工作台

- 导航入口命名为“缠论工作台”。
- 顶部提供代码/名称输入、来源状态和最后确认时间；支持从选股结果、自选股、个股详情带入标的。
- 四周期状态条固定展示日线、60m、30m、5m 的方向、最近中枢、背驰、买卖点和确认时间；点击后切换主图。
- 主图沿用当前 ECharts/K 线容器，不替换图表引擎。图层控件独立控制笔、线段、中枢、买卖点、背驰与既有 GSGF 证据。
- 右侧显示所选信号的参考段、背驰系数、状态、规则说明、失效条件和预警记录；底部显示回放时间轴与模拟订单队列。

### 图层视觉规则

- 笔：红绿细线，表示细粒度高低点连接。
- 线段：上涨蓝色、下跌紫色，粗于笔且优先级更高。
- 实中枢：低透明度实线边框和填充；虚中枢：更浅的虚线边框和填充。
- 买点绿色、卖点红色；标签为“类型 + 背驰系数”。
- 未确认信号以低透明度和 `?` 标识，确认信号为实色；所有状态不能只依赖颜色区分。
- 放大时展示笔和完整标签；缩小时收敛为线段、中枢和关键确认信号，避免 120 根以上 K 线发生标签遮挡。
- MACD 面板保留在主图下方，背驰参考段在主图与 MACD 上使用轻量连接与悬浮详情，避免大段文字覆盖价格。

个股详情页只显示简化缠论开关、四周期摘要与工作台入口；选股页只显示结构摘要、筛选器和跳转入口。

## 性能、缓存与后台任务

- 日线结构在候选池批量预计算；分钟结构仅刷新“日线通过初筛的候选 + 自选股 + 活跃订单标的”。
- 单标的工作台请求优先使用本地 bar/结构缓存。缓存未命中只为当前标的创建补数任务，不阻塞其它页面。
- 5m 缓存在当前 K 闭合前只读上一次确认结果；30m、60m 和日线按各自闭合边界刷新，避免页面刷新重复计算。
- 结构快照 key 包含 `symbol + period + last_closed_bar_at + adjustment_mode + rule_version`。任何一个键变化才允许重新计算。
- 预警和模拟盘消费结构快照 ID，而非重新在各自服务内计算指标。
- 候选池分钟任务保留并发上限、来源请求节流和单标的失败隔离；公开数据历史回填不允许高并发请求。

## 失败处理与风控

- `czsc` 无法导入：返回结构层 `unavailable`，K 线、GSGF、选股和竞价功能继续工作。
- TickFlow 实时失败：页面展示最后归档结构并标为 `stale`；禁用新预警和订单草案。
- 历史分钟 K 不足：显示样本不足与回填进度；不生成背驰、共振分或订单草案。
- 历史源异常/限流：记录来源错误、指数退避并允许手动重试；不把空响应缓存为有效数据。
- 价格跨除权不可对齐：标记 `adjustment_mismatch`，禁止跨缺口结构与相应订单。
- 订单草案或审批时，校验交易时段、数据新鲜度、信号确认状态、资金、已有持仓、单票仓位上限、单日交易次数、涨跌停、最大追价和信号过期时间。任一条件不满足进入 `rejected` 并记录原因。
- 模拟成交使用已记录报价与可配置滑点。回测成交只允许在触发信号后的下一根可交易 K 开盘价或明确的回放规则价格成交。

## 回放、回测与审计

- 回放按时间顺序逐根输入已闭合 K；每个时间点的结构计算不能读取后续 bar。
- 回测保存数据快照 ID、规则版本、周期组合、订单规则、滑点、初始资金和每笔订单状态。
- 模拟盘的人工审批动作与当时信号快照绑定；后续规则升级不改写旧订单原因。
- 报告至少按信号类型、背驰分类、周期共振组合和市场状态统计样本数、胜率、平均收益、盈亏比、最大回撤和未完成订单。

## 验收与测试

### 后端单元测试

- 交易时段聚合不跨午休，未闭合分钟 K 不进入确认结构。
- 包含关系、分型、笔、线段、实中枢、虚中枢和每类买卖点由固定 fixture 验证。
- 背驰系数、盘整背驰、顶背驰和底背驰保存正确参考段与中枢数量。
- 结构状态只能按 `observing -> provisional -> confirmed -> final` 合法迁移。
- 空、乱序、重复和复权不匹配数据不会生成可执行信号。
- 缓存 key 会在闭合 bar、规则版本或复权口径变化时失效。

### API 与集成测试

- TickFlow 5m/30m/60m 成功、权限不足、超时和空响应分别返回明确状态。
- TDX/mootdx 补数任务可分页写入、幂等重跑且不会覆盖已闭合快照。
- 工作台、选股摘要、预警去重、回放和模拟盘 API 契约稳定。
- 模拟订单不能绕过人工审批；所有风控拒绝条件可复现。
- 镜像构建与容器启动测试确认 `import czsc` 和其 OpenMP 动态库可用。

### 前端测试

- ECharts overlay 正确生成笔、线段、中枢、买卖点和背驰 series。
- 图层开关、周期切换、缩放收敛、键盘可达性和状态文案正确。
- 数据不足、回填中、过期和结构不可用状态不遮蔽原始 K 线。
- 工作台可由输入、选股和个股详情跳转；模拟订单仅能通过确认界面进入挂单状态。

### 无未来函数验证

对每个 fixture 和历史回放日期，分别以完整序列和截断到该 bar 的序列计算。截断计算产生的确认信号必须与完整序列在同一时点可见的信号一致；若完整序列导致前序结构被修订，前序信号只能保留为当时快照，不能提前作为最终标签或回测依据。

## 分阶段交付

### 第一阶段：结构基础与工作台

- 安装并验证 `czsc` 运行时。
- 建立分钟历史归档、TDX 补数、四周期结构分析 API 和 `/chanlun` 工作台。
- 完成主图基础图层、数据状态和回放输入。

### 第二阶段：完整图层、选股与预警

- 增加虚中枢、全系买卖点、背驰系数及图层缩放策略。
- 将结构摘要、筛选和共振分接入现有候选池与自选股。
- 增加确认信号的站内/现有通知通道预警和去重。

### 第三阶段：模拟盘与复盘

- 建立模拟账户、订单草案、人工审批、成交模拟、风控状态机和审计日志。
- 完成无未来函数回测、结构信号绩效报告和订单复盘。

每个阶段都必须独立通过数据源、测试和回放校验后，才进入下一阶段。
