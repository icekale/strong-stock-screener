# 金融工作台产品化 UI 重构设计

## 目标

将现有 Soybean 管理后台外观收敛为浅色机构金融终端，提升市场总览、竞价雷达、强势选股、个股 K 线和缠论工作台的扫描效率、状态可信度和移动端可用性，同时保留现有业务逻辑、数据接口和交易安全边界。

## 用户与设计原则

目标用户是需要反复扫描 A 股市场、确认竞价候选、检查 K 线与缠论结构并维护观察池的短线交易者。界面遵循以下原则：

- 先看市场方向和可执行信号，再看解释与维护信息。
- 视觉层级来自布局、字号和留白，不依赖渐变、发光和装饰。
- 红绿只表达涨跌和风险，所有重要状态同时显示文字。
- 数据源、模型版本、更新时间和失败原因必须可追溯。
- 保留高信息密度，但让列、数值、标签和动作有稳定位置。
- 页面失败时只隔离失败模块，不清空其他已经成功的数据。
- 桌面端与移动端使用不同的组合方式，不通过缩小字体压缩桌面布局。

## 视觉系统

### 色彩

采用 restrained 的浅色机构终端策略：

- 主色使用低饱和深蓝，用于主按钮、当前导航、链接和聚焦状态。
- 内容背景使用冷白灰，模块表面使用白色，边框使用低对比冷灰。
- 上涨/积极信号使用红色，下跌/风险使用绿色；标签和文本同时表达语义。
- 警告使用琥珀色，数据源失败使用明确的红色状态，不用单一颜色制造大面积状态卡。
- 不使用主视觉渐变、发光、玻璃拟态或大面积阴影。

### 字体与尺寸

- 使用现有系统无衬线字体栈，保持中文数据阅读稳定。
- 页面标题、模块标题、标签、正文、数字和辅助文字使用固定字号层级。
- 数据数字使用稳定的等宽或数字友好字形设置，价格、百分比、成交额避免原始浮点长串。
- 标题和描述保持短句，移除模板式解释文案和重复提示。

### 表面与间距

- 页面内容使用统一的内容内边距和模块间距。
- 卡片只用于真正独立的模块；同一模块内部优先使用分组、分隔线和留白，避免卡片嵌套卡片。
- 面板统一使用轻边框和 4-6px 圆角，hover/selected 状态通过背景和边框变化表达。
- 主操作按钮使用实色，次操作按钮使用默认样式，危险操作使用明确的文字和确认状态。

## 通用组件与页面骨架

新增或整理以下共享 UI 组件，页面只负责数据和业务动作：

- `PageHeader`：页面标题、短描述、日期/筛选/刷新等操作区。
- `MetricStrip`：统一的关键指标带，支持数字、状态、辅助比较值和加载态。
- `SectionHeader`：模块标题、数据来源、更新时间和模块级操作。
- `StatusTag`：统一状态文案、颜色、图标和可访问文本。
- `DataList`：统一列表密度、主次文本、右侧数值和空/失败状态。

页面骨架统一为：

```text
PageHeader
  -> 关键指标
  -> 主工作区
  -> 辅助信息
  -> 数据来源 / 更新时间
```

所有组件必须覆盖默认、hover、focus、active、disabled、loading、empty 和 error 状态；不改变已有 API 数据结构。

## AppShell 与导航

- 侧边栏按交易工作流排序：市场总览、竞价雷达、强势选股、板块热图、个股/缠论、自选风险、情绪复盘、模型与数据源。
- 当前导航改为浅蓝背景加左侧深蓝指示条，移除整块紫色高亮。
- 收起侧边栏后保留稳定图标尺寸、悬浮提示和可恢复展开入口。
- 顶栏保留面包屑、当前工作区和必要工具，减少模板无关入口。
- 标签栏保留已打开的股票/工作区，但使用细边框和明确关闭动作，降低装饰感。
- 移动端使用抽屉导航，滚动内容不被侧栏或固定 footer 遮挡。
- 页面切换只保留短促、可取消的淡入效果，并尊重 reduced-motion 设置。

## 核心页面布局

### 市场总览

首屏顺序为主要指数、板块资金流/盘中情绪，然后是成交额、涨跌家数、涨停跌停和盘面状态。竞价 Top3 与市场关注榜作为次级决策区，数据来源和更新时间收在页面底部。

### 竞价雷达

顶部只保留交易日、刷新、模型状态和关键指标。Top3 使用紧凑优先级列表，竞价强度榜改为高密度可扫描表格，固定显示开盘涨幅、成交额、行业、信号、风险和加入自选动作。

### 强势选股

筛选条件集中在单行工具栏。结果采用稳定列对齐的列表/表格，固定显示评分、缠论共振、数据完整性、风险原因和操作入口。模型解释改为详情展开，避免挤压候选列表。

### 个股 K 线

报价摘要和周期切换固定在图表上方，K 线占主要宽度。均线、副图、GSGF、缠论等控制收敛到一条工具栏。信息、战法和概念使用页签切换，减少内部嵌套卡片，保留多图、指标数值和结构标注能力。

### 缠论工作台

代码、周期、图层和指标控制集中在顶部。K 线和结构绘图区占主要空间，信号、背驰、中枢和人工确认模拟盘在下方两栏展示，突出当前状态、风险原因和人工确认动作。

### 辅助页面

板块热图突出图表、筛选和成分股；情绪复盘突出交易许可和盘中提醒；自选风险突出观察池操作和结构触发；模型与数据源突出数据源健康、任务状态、缓存维护和模型维护包。

## 状态、数据和响应式行为

- 每个数据模块统一处理读取中、成功、部分成功、无数据和失败状态。
- 数据源状态同时显示来源、状态和时间；失败原因靠近对应模块。
- 空状态提供下一步动作，例如选择板块、运行模型或检查数据源。
- 桌面端采用双栏和表格列布局，平板收缩为主栏，手机改为单栏和两行信息块。
- 关键数值在移动端保持高对比和可扫描位置，不能通过缩小字号解决溢出。
- 保留现有选股、竞价、K 线、缠论、模拟盘和数据源接口，不引入新的业务数据依赖。

## 实施顺序

1. 整理全局 token、AppShell、导航、通用页面头和状态组件。
2. 改造市场总览、竞价雷达和强势选股，验证核心扫描工作流。
3. 改造个股 K 线和缠论工作台，验证图表控制、指标和模拟盘行为不变。
4. 用同一套组件改造板块热图、情绪复盘、自选风险和模型与数据源。
5. 每批执行前端单元测试、类型检查、生产构建和桌面/移动端浏览器回归。

## 非目标

- 不更换 Soybean/Vue/Ant Design 技术栈。
- 不迁移到另一个后台模板。
- 不修改后端 API、数据源、模型算法或交易执行逻辑。
- 不增加营销式首页、装饰插画或无业务价值的动画。
- 不在本轮重做缠论计算、选股逻辑或竞价模型。
