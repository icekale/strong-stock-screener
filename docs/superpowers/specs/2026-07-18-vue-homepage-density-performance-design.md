# Vue 首页信息密度与性能设计

## 目标

重构生产环境使用的 Vue 首页 `/`，移除竞价 Top3，提高首屏信息密度，并减少首次进入、页面返回和并发刷新产生的等待与重复请求。改动只落在 `apps/web-vue`；保留竞价页面内的 Top3、后端接口和业务计算语义。

## 已确认布局

首页采用“紧凑决策台”布局，顺序如下：

1. 页面标题、实时交易日和刷新操作。
2. 四个主要指数，使用紧凑横向单元展示价格与涨跌幅。
3. 四项全市场状态：总成交额、上涨/下跌、涨停/跌停、盘面状态。
4. 主要分析区采用约 `60 / 40` 双栏：左侧板块净流入柱状图，右侧板块实时曲线。
5. 底部展示市场关注榜与紧凑的数据源状态。

竞价 Top3 不请求、不渲染，也不保留首页占位。桌面维持双栏；窄屏按指数、指标、图表、榜单、数据状态依次退化为单栏或双列。

## 板块实时曲线

- 复用板块页面现有的 `getSectorReplicaRadar` 和 `buildSectorReplicaChartOption`，不新建近似数据或绘图算法。
- 默认请求 `mode=strength`、`limit=5`、`stock_limit=1`，仅携带首页需要的五条曲线和最小成分股载荷。
- 图表右上角保留“强度 / 主力流”分段切换；切换只刷新实时曲线，不刷新其他首页面板。
- 图表使用 `compact=true`，高度固定，图例允许换行但不得覆盖绘图区。
- 左侧板块资金流继续使用 `getSectorRadar`，两张图分别回答“当前净流入排名”和“盘中板块强弱演变”，避免重复表达。

## 请求与状态架构

首页拆成四个独立资源：

| 资源 | 请求 | 首屏职责 |
| --- | --- | --- |
| 市场概览 | `getMarketOverview()` | 指数与全市场状态，最高优先级 |
| 市场排行 | `getMarketRankings(12)` | 底部关注榜 |
| 板块资金流 | `getSectorRadar(12)` | 左侧资金流柱状图 |
| 板块实时曲线 | `getSectorReplicaRadar(...)` | 右侧实时曲线 |

四个请求挂载后并行启动，但分别维护 `loading`、`refreshing`、`error` 和现有数据。市场概览返回后立即提交指数与状态，不等待其他请求。单个次要资源失败只影响对应面板；已有数据在刷新失败时保留并标记为旧数据。底部数据状态合并并按来源去重市场概览、板块资金流和板块实时曲线实际返回的 `source_status`。

页面日期控件不再驱动与日期无关的实时接口。标题区展示响应中的交易日和最近更新时间，避免用户误以为首页支持历史切换。

## 短时缓存与并发去重

新增一个无 Vue 依赖的内存请求缓存协调器：

- 默认 TTL 为 15 秒，缓存键包含面板、模式、limit 和其他请求参数。
- 同一键存在进行中的请求时复用同一个 Promise。
- 页面离开后返回时，在 TTL 内立即使用成功结果，不写入持久化存储。
- 手动刷新绕过已完成缓存；若同键请求仍在进行中，则复用该请求，避免连点制造并发风暴。
- 失败不覆盖成功缓存，也不写入可命中的失败结果。

该缓存只协调浏览器请求，不修改后端缓存、数据源优先级或实时曲线采样方式。

## 图表加载

- `MarketTrendChart` 不再出现在首页。
- 首页图表组件使用 `defineAsyncComponent` 动态导入；文字指数和市场状态先进入 DOM，下一帧再挂载图表容器。
- ECharts 类型使用 type-only import，不进入首屏运行时代码。
- 图表容器具有稳定高度，异步组件加载和空状态不会引发布局跳动。
- 不增加自动轮询；实时性沿用后端数据更新和用户刷新，避免首页常驻产生额外压力。

## 错误处理

- 市场概览首次加载失败时显示页面级警告，其余面板仍可独立呈现。
- 板块资金流、曲线或排行失败时，在原面板内显示简短错误和重试，不清空其他成功结果。
- 曲线响应为空时显示“等待板块采样”，并保留强度/主力流切换。
- 数据状态只显示实际返回的数据源，不生成虚假的“正常”状态。

## 测试与验收

1. 缓存单元测试覆盖 TTL 命中、过期、并发去重、强制刷新和失败保留旧值。
2. 首页组件测试验证不再导入或调用 Top3，四项资源独立更新，模式切换只请求板块曲线。
3. API 请求测试验证板块实时曲线使用 `limit=5` 与 `stock_limit=1`。
4. 运行 `pnpm test:unit`、`pnpm typecheck` 和 `pnpm build`。
5. 浏览器检查桌面与移动视口：首屏无大块空白、图例不重叠、无横向滚动、Top3 不出现。
6. 网络检查：缓存窗口内离开并返回首页不重复请求；手动刷新会请求新数据；ECharts 在文字首屏之后加载。

## 不在本次范围

- 不修改竞价页面、Top3 模型或训练逻辑。
- 不改造后端板块算法、qxlive 数据源或采样器。
- 不引入新的状态管理、请求库或持久化行情缓存。
- 不修改个股 K 线、缠论和其他工作台页面；这些性能优化在首页稳定后单独处理。
