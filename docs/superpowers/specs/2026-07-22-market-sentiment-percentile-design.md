# 市场情绪百分位模型设计

## 1. 目标

在现有 `/sentiment`“情绪与复盘”页面增加一个收盘后更新的市场情绪百分位模型，用可解释的 `0-100` 分数回答：当前 A 股市场相对自身过去约两年处于冰点、偏冷、中性、偏热还是过热状态。

模型只使用用户确认的五个数据维度：

1. 量能百分位；
2. 5 日指数波动；
3. 股价位置；
4. 5 日股价振幅；
5. 量能趋势。

五项等权，各占 `20%`。模型是描述当前市场温度的统计指标，不以未来收益训练权重，不替代现有短线情绪、交易许可、盘中提醒或人工交易判断。

## 2. 已确认决策

- 市场基准使用中证全指 `000985.SH`；
- “5 日指数波动”定义为 5 日累计涨跌幅，而不是收益率标准差；
- 5 日振幅结合 5 日涨跌方向，避免把恐慌下跌误判为过热；
- 量能趋势定义为 5 日平均成交额相对 20 日平均成交额；
- 每个历史日期使用截至该日的滚动 500 个交易日，不使用未来数据；
- 五因子初始权重均为 `20%`；
- 正式分数只使用已完成日 K，每日收盘后定稿；
- 页面展示最近 500 个完整综合分，约两年走势；
- `joinylee/stock-sentiment-cn` 不作为代码或模型依赖。该仓库没有本设计所需的滚动五因子实现，且存在模拟值、硬编码凭证、缺失运行依赖和许可证文件不完整等风险。

## 3. 数据来源与时间规则

### 3.1 主数据源

主数据源使用现有 TickFlow 日 K Provider，一次读取中证全指约 1020 条日 K。已在目标 Unraid 环境验证 `000985.SH` 可返回 1020 条记录，覆盖 `2022-05-11` 至 `2026-07-22`，包含开盘、最高、最低、收盘和成交额。

1020 条原始记录的用途：

- 前约 520 条用于构建最早展示点的 500 日因子基线和 20 日均量预热；
- 后 500 条形成页面展示的最近两年情绪曲线。

### 3.2 已完成交易日

服务按 `Asia/Shanghai` 判断交易日：

- 交易日 `15:10` 前排除当日日 K；
- `15:10` 后，只有当日 OHLC 和成交额均有效时才纳入；
- 周末、节假日和数据源延迟时使用最近完整交易日；
- 不外推盘中成交额，不生成盘中正式百分位。

### 3.3 数据质量

输入记录按日期升序排列并去重。价格必须为正，最高价不得低于最低价，成交额必须为正。模型只输出五个因子全部有效的综合分，不对缺失因子重新归一化权重。

历史不足或最新点不完整时：

- 有最近成功缓存：返回缓存并标记为过期；
- 无缓存：返回“样本不足”或具体数据错误，不生成替代分数；
- 不缩短 500 日百分位窗口冒充正式结果。

## 4. 五因子模型

### 4.1 百分位算法

除股价位置外，其余原始因子使用截至当前日期的最近 500 个同类原始值计算经验百分位。并列值使用中间秩：

```text
百分位 = (小于当前值的样本数 + 0.5 * 等于当前值的样本数) / 500 * 100
```

当前值包含在自己的滚动窗口中。结果限制在 `0-100` 并保留一位小数。每个日期的计算只访问该日期及之前的数据。

### 4.2 量能百分位

```text
量能原始值[t] = amount[t]
量能得分[t] = amount[t] 在最近 500 个成交额中的经验百分位
```

使用成交额而不是成交量，避免指数成分和计量单位变化降低跨期可比性。

### 4.3 5 日指数波动

按用户确认的口径，“波动”表示有方向的 5 日累计涨跌幅：

```text
5日涨跌[t] = close[t] / close[t-5] - 1
5日指数得分[t] = 5日涨跌[t] 在最近 500 个 5 日涨跌值中的经验百分位
```

上涨越强得分越高，下跌越强得分越低。

### 4.4 股价位置

```text
500日最低价[t] = min(low[t-499:t])
500日最高价[t] = max(high[t-499:t])
股价位置[t] = (close[t] - 500日最低价[t])
              / (500日最高价[t] - 500日最低价[t]) * 100
```

分母为零时该因子不可用。结果限制在 `0-100`。该因子直接表示指数在两年价格区间中的位置，不再进行第二次百分位转换。

### 4.5 5 日方向振幅

```text
5日振幅[t] = (max(high[t-4:t]) - min(low[t-4:t])) / close[t-5]
方向[t] = sign(5日涨跌[t])
方向振幅[t] = 方向[t] * 5日振幅[t]
振幅得分[t] = 方向振幅[t] 在最近 500 个方向振幅值中的经验百分位
```

5 日涨跌为零时方向为零。大振幅上涨提高分数，大振幅下跌降低分数。

### 4.6 量能趋势

```text
量能趋势[t] = MA5(amount)[t] / MA20(amount)[t] - 1
趋势得分[t] = 量能趋势[t] 在最近 500 个量能趋势值中的经验百分位
```

`> 0` 表示近期量能高于中期基线，`< 0` 表示近期缩量。

### 4.7 综合分和等级

```text
情绪百分位 = (
  量能得分
  + 5日指数得分
  + 股价位置
  + 振幅得分
  + 量能趋势得分
) / 5
```

等级固定为：

| 分数 | 等级 |
| --- | --- |
| `< 20` | 冰点 |
| `20 <= score < 40` | 偏冷 |
| `40 <= score < 60` | 中性 |
| `60 <= score < 80` | 偏热 |
| `>= 80` | 过热 |

阈值属于产品展示分层，不表述为未来涨跌概率或买卖信号。

## 5. 服务边界与持久化

新增独立的市场情绪百分位服务，职责分为：

- 日 K Provider：读取中证全指历史日 K；
- Calculator：校验输入、计算五因子、滚动百分位和综合分；
- Store：原子读写最近成功快照；
- Service：过滤未完成日 K、决定缓存命中、刷新和过期降级；
- Validation：离线统计不同情绪分层的后续市场表现，不修改模型权重。

持久化目录：

```text
/app/data/sentiment-percentile/
  latest.json
  validation-v1.json
```

`latest.json` 保存：模型版本、基准代码、最新完整交易日、五项权重、最近 500 个完整点、来源状态和生成时间。写入使用临时文件加原子替换，避免容器退出留下半文件。

模型版本从 `market-sentiment-percentile-v1` 开始。公式、窗口、权重或阈值变化必须提升版本，不静默重算旧缓存。

## 6. API 契约

新增接口：

```http
GET /api/short-term/sentiment/percentile?as_of=2026-07-22&refresh=false
```

参数：

- `as_of` 可选：返回不晚于该日期的选中点和历史切片；默认使用最新完整交易日；
- `refresh` 默认 `false`：`true` 时强制检查数据源并重算，失败后仍可降级到最近成功缓存。

响应包含：

- `model_version`、`benchmark_symbol`、`benchmark_name`；
- `window_size=500` 和五项固定权重；
- `latest_complete_trade_date`、`selected_trade_date`；
- `selected`：综合分、等级、五项因子分和原始值；
- `history`：最多 500 个完整点，每点包含综合分、等级和五项因子分；
- `cache_status`：`fresh`、`cached`、`stale`；
- `source_status`、`generated_at`、`notes`。

历史 `as_of` 只切片已经计算完成的序列，不用该日期之后的数据重算历史点。超出缓存历史范围时返回明确空状态，不伪造早期结果。

## 7. 更新与性能

- 同一最新完整交易日优先读取持久化缓存；
- 收盘后首次访问发现数据日期前进时，读取约 1020 条日 K 并重算；
- 页面刷新按钮通过 `refresh=true` 强制检查；
- 单次计算规模约为 1020 条记录和 500 日窗口，不需要 NumPy、机器学习库或 GPU；
- API 内存缓存与持久化缓存共同避免页面重复抓取；
- 数据源失败不得清空最近成功缓存；
- 第一版不增加独立定时任务，页面按需刷新即可完成每日更新。

## 8. 页面信息架构

新增模块位于 `/sentiment` 页面标题之后、现有短线情绪指标之前。它是较慢的收盘市场温度层；原有涨停、炸板、交易许可、主线信号和盘中提醒保持原口径与顺序。

### 8.1 桌面布局

模块采用一块完整工作区：

- 左侧约三分之二：最近两年情绪折线图；
- 右侧约三分之一：当前综合分、等级和五条紧凑因子刻度；
- 标题显示“市场情绪百分位”、中证全指、收盘模型、数据日期和缓存状态。

不使用雷达图。五条水平刻度更便于比较数值，也不会把不同因子的含义伪装成面积。

### 8.2 情绪走势图

- Y 轴固定 `0-100`；
- `0-20` 使用克制的淡绿背景并标注“冰点区”；
- `80-100` 使用克制的淡红背景并标注“过热区”；
- `20` 和 `80` 显示水平边界线；
- 普通历史点不逐点绘制圆点；
- 仅冰点、过热和最新点显示强调标记；
- 悬浮提示显示日期、综合分、等级、五项因子分及其原始值；
- 最新点单独标注，但颜色仍遵循其实际等级，不把偏冷点误画为过热点。

### 8.3 当前因子区

右侧显示：

- 当前综合分和“冰点 / 偏冷 / 中性 / 偏热 / 过热”；
- 量能百分位；
- 5 日指数涨跌；
- 股价位置；
- 5 日方向振幅；
- 量能趋势。

每项同时显示因子分和原始值，例如“18.4 · 5 日 -3.12%”。颜色不是唯一含义，数值和文字必须始终存在。

### 8.4 交互与响应式

- 页面日期选择器变化时，高亮不晚于所选日期的历史点；
- 点击或键盘选择图上日期后，右侧因子区同步更新；
- 点击刷新只刷新该模块和现有情绪请求，不清空已有成功数据；
- 窄屏改为走势图在上、因子区在下；
- 图表使用稳定高度，加载、错误和文字变化不得引起明显布局跳动；
- 图表关闭无意义的长动画，并支持 `prefers-reduced-motion`；
- 390px 宽度下页面不产生横向溢出。

前端复用现有 `EChart`、工作台 token、字体、红涨绿跌规则和错误组件，不引入新的图表依赖或独立页面。

## 9. 错误与降级

- 百分位接口失败：新模块显示单独错误，现有情绪页面继续可用；
- 缓存过期：保留曲线和因子值，在标题标明“截至 YYYY-MM-DD”；
- 当前选择日期无结果：曲线保留，因子区显示该日期不可用；
- 历史不足：显示所需与实际样本数，不按短窗口计算；
- 单个因子缺失：该日期不产生综合分，不把其余四项重新放大到 100%；
- 数据源恢复后，刷新成功替换过期状态。

## 10. 历史验证

提供可复现的离线验证命令，使用与线上完全相同的 Calculator，对每个历史日期执行 walk-forward 计算。验证只评估，不训练权重。

报告按五档情绪区间统计：

- 样本数量；
- 未来 5、10、20 日平均和中位收益；
- 未来 5、10、20 日上涨比例；
- 未来 5、10、20 日最大回撤；
- 冰点和过热状态的平均持续天数。

未来收益只作为历史点之后的验证标签，不进入该历史点的模型输入。报告必须同时输出样本不足提示，不能根据少量极端样本宣称模型有效。

如果各情绪区间没有稳定区分度，线上仍可将该分数用于描述市场温度，但产品文案必须标注“择时有效性未验证”，不得自动调整等权权重掩盖结果。

## 11. 测试与验收

### 11.1 后端单元测试

- 五项原始公式和等权综合分；
- 500 日窗口的首个有效点；
- 经验百分位中间秩和并列值；
- 5 日上涨、下跌和零涨跌的方向振幅；
- 500 日价格区间分母为零；
- 缺失或非法 OHLC、成交额；
- 任意未来输入变化不影响该日及之前结果；
- 未收盘当日日 K 过滤；
- 缓存命中、版本失配、过期回退和原子写入；
- 离线验证标签不进入模型计算。

### 11.2 API 和前端测试

- API 默认日期、历史 `as_of`、强制刷新和错误契约；
- 页面独立加载百分位、短线情绪、交易许可和盘中提醒；
- 百分位接口失败不清空其它模块；
- 图表固定显示 `0-100`、冰点区、过热区和最新点；
- 选择历史日期更新五项因子；
- stale 状态显示实际缓存日期；
- 因子名称、分数、原始值和模型说明完整；
- 键盘操作和非颜色状态表达可用。

### 11.3 发布验收

- Python 全量测试通过；
- Vue 全量测试、类型检查和生产构建通过；
- 本地与 Unraid 的桌面和 390px 窄屏截图无重叠、截断或页面横向溢出；
- 目标环境实际返回至少 1000 条中证全指有效日 K；
- 缓存命中时接口快速返回，普通页面刷新不重复拉取日 K；
- Docker 单容器健康，`/sentiment` 和百分位 API 返回 200。

## 12. 非目标

- 不训练机器学习、PCA 或预测模型；
- 不使用未来收益调整五项权重；
- 不提供盘中估算百分位；
- 不把新分数混入现有 `emotion_score` 或交易许可；
- 不触发模拟下单、自动下单或仓位调整；
- 不新增独立情绪路由；
- 不复制 `stock-sentiment-cn` 的报告脚本、固定评分、通知凭证或外部 AI 依赖；
- 不在本期增加社交媒体情绪、北向资金、龙虎榜或新闻因子。
