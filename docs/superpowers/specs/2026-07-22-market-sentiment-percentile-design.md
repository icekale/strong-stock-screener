# 市场情绪百分位模型设计

## 1. 目标

在现有 `/sentiment`“情绪与复盘”页面增加一个收盘后更新的市场情绪百分位模型，用可解释的 `0-100` 分数回答：当前 A 股市场相对自身过去约两年处于冰点、偏冷、中性、偏热还是过热状态。每个交易日的正式分数生成后，必须调用一次配置好的 OpenAI 兼容 LLM，结合五因子和现有市场结构化数据生成盘后解读。

模型只使用用户确认的五个数据维度：

1. 量能百分位；
2. 5 日指数波动；
3. 股价位置；
4. 5 日股价振幅；
5. 量能趋势。

五项等权，各占 `20%`。统计模型不以未来收益训练权重；LLM 只解释已经确定的分数、市场证据和风险姿态，不得修改权重、等级或现有交易许可。

## 2. 已确认决策

- 市场基准使用中证全指 `000985.SH`；
- “5 日指数波动”定义为 5 日累计涨跌幅，而不是收益率标准差；
- 5 日振幅结合 5 日涨跌方向，避免把恐慌下跌误判为过热；
- 量能趋势定义为 5 日平均成交额相对 20 日平均成交额；
- 每个历史日期使用截至该日的滚动 500 个交易日，不使用未来数据；
- 五因子初始权重均为 `20%`；
- 正式分数只使用已完成日 K，每日收盘后定稿；
- 页面展示最近 500 个完整综合分，约两年走势；
- 每个交易日必须使用 OpenAI 兼容接口生成一份 LLM 盘后解读；
- LLM 输入包括五因子和现有涨跌家数、涨跌停、炸板、封板率、连板高度、成交额、主线板块及历史分层统计；
- LLM 输出包含市场结论、主要驱动、因子背离、历史位置、风险姿态、次日观察条件和风险提示；
- 风险姿态只允许“进攻 / 平衡 / 防守 / 等待”，不得包含具体股票或仓位比例；
- `joinylee/stock-sentiment-cn` 不作为代码或模型依赖。该仓库没有本设计所需的滚动五因子实现，且存在模拟值、硬编码凭证、缺失运行依赖和许可证文件不完整等风险。

## 3. 数据来源与时间规则

### 3.1 主数据源

主数据源使用现有 TickFlow 日 K Provider，一次读取中证全指约 1020 条日 K。已在目标 Unraid 环境验证 `000985.SH` 可返回 1020 条记录，覆盖 `2022-05-11` 至 `2026-07-22`，包含开盘、最高、最低、收盘和成交额。

1020 条原始记录的用途：

- 前约 520 条用于构建最早展示点的 500 日因子基线和 20 日均量预热；
- 后 500 条形成页面展示的最近两年情绪曲线。

### 3.2 已完成交易日

服务按 `Asia/Shanghai` 判断交易日：

- 交易日 `15:10` 前排除当日日 K；
- `15:10` 后，只有当日 OHLC 和成交额均有效时才纳入；
- 周末、节假日和数据源延迟时使用最近完整交易日；
- 不外推盘中成交额，不生成盘中正式百分位。

### 3.3 数据质量

输入记录按日期升序排列并去重。价格必须为正，最高价不得低于最低价，成交额必须为正。模型只输出五个因子全部有效的综合分，不对缺失因子重新归一化权重。

历史不足或最新点不完整时：

- 有最近成功缓存：返回缓存并标记为过期；
- 无缓存：返回“样本不足”或具体数据错误，不生成替代分数；
- 不缩短 500 日百分位窗口冒充正式结果。

## 4. 五因子模型

### 4.1 百分位算法

除股价位置外，其余原始因子使用截至当前日期的最近 500 个同类原始值计算经验百分位。并列值使用中间秩：

```text
百分位 = (小于当前值的样本数 + 0.5 * 等于当前值的样本数) / 500 * 100
```

当前值包含在自己的滚动窗口中。结果限制在 `0-100` 并保留一位小数。每个日期的计算只访问该日期及之前的数据。

### 4.2 量能百分位

```text
量能原始值[t] = amount[t]
量能得分[t] = amount[t] 在最近 500 个成交额中的经验百分位
```

使用成交额而不是成交量，避免指数成分和计量单位变化降低跨期可比性。

### 4.3 5 日指数波动

按用户确认的口径，“波动”表示有方向的 5 日累计涨跌幅：

```text
5日涨跌[t] = close[t] / close[t-5] - 1
5日指数得分[t] = 5日涨跌[t] 在最近 500 个 5 日涨跌值中的经验百分位
```

上涨越强得分越高，下跌越强得分越低。

### 4.4 股价位置

```text
500日最低价[t] = min(low[t-499:t])
500日最高价[t] = max(high[t-499:t])
股价位置[t] = (close[t] - 500日最低价[t])
              / (500日最高价[t] - 500日最低价[t]) * 100
```

分母为零时该因子不可用。结果限制在 `0-100`。该因子直接表示指数在两年价格区间中的位置，不再进行第二次百分位转换。

### 4.5 5 日方向振幅

```text
5日振幅[t] = (max(high[t-4:t]) - min(low[t-4:t])) / close[t-5]
方向[t] = sign(close[t] - close[t-1])
方向振幅[t] = 方向[t] * 5日振幅[t]
振幅得分[t] = 方向振幅[t] 在最近 500 个方向振幅值中的经验百分位
```

5 日涨跌为零时方向为零。大振幅上涨提高分数，大振幅下跌降低分数。

### 4.6 量能趋势

```text
量能趋势[t] = MA5(amount)[t] / MA20(amount)[t] - 1
趋势得分[t] = 量能趋势[t] 在最近 500 个量能趋势值中的经验百分位
```

`> 0` 表示近期量能高于中期基线，`< 0` 表示近期缩量。

### 4.7 综合分和等级

```text
情绪百分位 = (
  量能得分
  + 5日指数得分
  + 股价位置
  + 振幅得分
  + 量能趋势得分
) / 5
```

等级固定为：

| 分数 | 等级 |
| --- | --- |
| `< 20` | 冰点 |
| `20 <= score < 40` | 偏冷 |
| `40 <= score < 60` | 中性 |
| `60 <= score < 80` | 偏热 |
| `>= 80` | 过热 |

阈值属于产品展示分层，不表述为未来涨跌概率或买卖信号。

## 5. 服务边界与持久化

新增独立的市场情绪百分位服务，职责分为：

- 日 K Provider：读取中证全指历史日 K；
- Calculator：校验输入、计算五因子、滚动百分位和综合分；
- Store：原子读写最近成功快照；
- Service：过滤未完成日 K、决定缓存命中、刷新和过期降级；
- Validation：离线统计不同情绪分层的后续市场表现，不修改模型权重。
- Analysis Input Builder：把正式五因子结果、现有市场情绪快照和历史验证摘要整理为稳定 JSON；
- LLM Analyzer：复用现有 `ai_analysis` OpenAI 兼容配置和 JSON 响应解析，执行严格的结构化盘后分析；
- Analysis Store：按交易日、模型版本、LLM 模型和输入哈希持久化成功结果及失败状态；
- Daily Sampler：应用启动后在 `15:15 Asia/Shanghai` 起检查最新完整交易日，缺少成功解读时触发生成，直至成功或记录可重试失败。

持久化目录：

```text
/app/data/sentiment-percentile/
  latest.json
  validation-v1.json
  analysis/
    YYYY-MM-DD.json
```

`latest.json` 保存：模型版本、基准代码、最新完整交易日、五项权重、最近 500 个完整点、来源状态和生成时间。写入使用临时文件加原子替换，避免容器退出留下半文件。

模型版本从 `market-sentiment-percentile-v1` 开始。当前实现为 `market-sentiment-percentile-v2`：5 日振幅仍使用 5 日价格范围和 5 日前收盘价归一化，但方向改为当日收盘相对前一交易日收盘的方向，使收盘后的市场普跌能够及时反映。公式、窗口、权重或阈值变化必须提升版本，不静默重算旧缓存。

LLM 分析记录保存统计模型版本、LLM provider、LLM 模型名、输入哈希、请求时间、完成时间、尝试次数、状态和结构化结果。相同输入哈希已有成功结果时不得重复调用。

## 6. API 契约

新增接口：

```http
GET /api/short-term/sentiment/percentile?as_of=2026-07-22&refresh=false
```

参数：

- `as_of` 可选：返回不晚于该日期的选中点和历史切片；默认使用最新完整交易日；
- `refresh` 默认 `false`：`true` 时强制检查数据源并重算，失败后仍可降级到最近成功缓存。

响应包含：

- `model_version`、`benchmark_symbol`、`benchmark_name`；
- `window_size=500` 和五项固定权重；
- `latest_complete_trade_date`、`selected_trade_date`；
- `selected`：综合分、等级、五项因子分和原始值；
- `history`：最多 500 个完整点，每点包含综合分、等级和五项因子分；
- `cache_status`：`fresh`、`cached`、`stale`；
- `source_status`、`generated_at`、`notes`。

历史 `as_of` 只切片已经计算完成的序列，不用该日期之后的数据重算历史点。超出缓存历史范围时返回明确空状态，不伪造早期结果。

新增 LLM 分析接口：

```http
GET /api/short-term/sentiment/percentile/analysis?trade_date=2026-07-22
POST /api/short-term/sentiment/percentile/analysis/generate?trade_date=2026-07-22&force=false
```

`GET` 返回 `not_generated`、`unconfigured`、`pending`、`ready` 或 `failed` 状态以及缓存结果。`POST` 在正式百分位齐全后生成分析；`force=true` 只用于人工重试或输入数据已修正的情况。相同输入成功后普通刷新不得再次消耗 LLM 请求。

五因子及其原始值是生成分析的硬性输入。涨跌家数、涨跌停、炸板、封板率、连板高度、成交额、主线板块和历史验证摘要按各自数据日期写入，允许字段明确为 `null` 或 `unavailable`；单个辅助字段缺失不得阻断当天分析，但 LLM 必须在输出中说明数据质量，不得补造数值。输入使用字段排序后的规范 JSON 计算 SHA-256，保证缓存去重可复现。

分析响应至少包含：

- `market_conclusion`：一段简短市场结论；
- `key_drivers`：2-4 条带数值依据的主要驱动；
- `factor_divergence`：因子一致、背离或数据不足说明；
- `historical_context`：当前分层与历史样本的关系；
- `risk_posture`：`attack`、`balanced`、`defensive`、`wait`；
- `next_session_watch`：2-4 条可验证的次日观察条件；
- `risk_note`：不构成买卖建议的风险说明；
- provider、模型、输入哈希、交易日和生成时间。

## 7. 更新与性能

- 同一最新完整交易日优先读取持久化缓存；
- 收盘后首次访问发现数据日期前进时，读取约 1020 条日 K 并重算；
- 页面刷新按钮通过 `refresh=true` 强制检查；
- 单次计算规模约为 1020 条记录和 500 日窗口，不需要 NumPy、机器学习库或 GPU；
- API 内存缓存与持久化缓存共同避免页面重复抓取；
- 数据源失败不得清空最近成功缓存；
- 正式五因子分数齐全后触发一次 LLM 分析；辅助市场字段缺失时保留数据质量状态；成功结果按输入哈希缓存；
- LLM 使用现有 `ai_analysis` 的 Base URL、API Key 和模型配置，不新增第二套密钥；
- LLM 请求使用低温度和 JSON 输出，单次输出控制在约 500-800 个中文字符；
- 新增与现有 sampler 生命周期一致的轻量后台线程，每 5 分钟检查一次，在交易日 `15:15` 后为最新完整交易日补生成；同一输入哈希成功后不再调用；
- 容器在收盘后才启动或定时检查失败时，首次百分位请求只触发后台补生成并立即返回统计结果，普通页面请求不等待外部 LLM；
- `unconfigured` 或连续 3 次失败表示当天 LLM 日更要求尚未满足；失败后至少冷却 30 分钟再由后台发起下一轮，页面同时保留人工重试入口。

## 8. 页面信息架构

新增模块位于 `/sentiment` 页面标题之后、现有短线情绪指标之前。它是较慢的收盘市场温度层；原有涨停、炸板、交易许可、主线信号和盘中提醒保持原口径与顺序。

### 8.1 桌面布局

模块采用一块完整工作区：

- 左侧约三分之二：最近两年情绪折线图；
- 右侧约三分之一：当前综合分、等级和五条紧凑因子刻度；
- 标题显示“市场情绪百分位”、中证全指、收盘模型、数据日期和缓存状态。

不使用雷达图。五条水平刻度更便于比较数值，也不会把不同因子的含义伪装成面积。

### 8.2 情绪走势图

- Y 轴固定 `0-100`；
- `0-20` 使用克制的淡绿背景并标注“冰点区”；
- `80-100` 使用克制的淡红背景并标注“过热区”；
- `20` 和 `80` 显示水平边界线；
- 普通历史点不逐点绘制圆点；
- 仅冰点、过热和最新点显示强调标记；
- 悬浮提示显示日期、综合分、等级、五项因子分及其原始值；
- 最新点单独标注，但颜色仍遵循其实际等级，不把偏冷点误画为过热点。

### 8.3 当前因子区

右侧显示：

- 当前综合分和“冰点 / 偏冷 / 中性 / 偏热 / 过热”；
- 量能百分位；
- 5 日指数涨跌；
- 股价位置；
- 5 日方向振幅；
- 量能趋势。

每项同时显示因子分和原始值，例如“18.4 · 5 日 -3.12%”。颜色不是唯一含义，数值和文字必须始终存在。

### 8.4 交互与响应式

- 页面日期选择器变化时，高亮不晚于所选日期的历史点；
- 点击或键盘选择图上日期后，右侧因子区同步更新；
- 点击刷新只刷新该模块和现有情绪请求，不清空已有成功数据；
- 窄屏改为走势图在上、因子区在下；
- 图表使用稳定高度，加载、错误和文字变化不得引起明显布局跳动；
- 图表关闭无意义的长动画，并支持 `prefers-reduced-motion`；
- 390px 宽度下页面不产生横向溢出。

前端复用现有 `EChart`、工作台 token、字体、红涨绿跌规则和错误组件，不引入新的图表依赖或独立页面。

### 8.5 AI 盘后解读

走势图和当前因子区下方增加“AI 盘后解读”，不使用聊天气泡或聊天输入框。区域展示：

- 市场结论和风险姿态；
- 2-4 条主要驱动；
- 因子背离和历史位置；
- 2-4 条次日观察条件；
- LLM 模型、生成时间和风险提示。

市场结论和风险姿态处于第一视觉层级，其余内容使用紧凑分组。每条驱动和观察条件必须包含输入中的具体数字、等级或阈值，不展示模型思维链。

## 9. 错误与降级

- 百分位接口失败：新模块显示单独错误，现有情绪页面继续可用；
- 缓存过期：保留曲线和因子值，在标题标明“截至 YYYY-MM-DD”；
- 当前选择日期无结果：曲线保留，因子区显示该日期不可用；
- 历史不足：显示所需与实际样本数，不按短窗口计算；
- 单个因子缺失：该日期不产生综合分，不把其余四项重新放大到 100%；
- 数据源恢复后，刷新成功替换过期状态。
- LLM 未配置：显示“AI 分析未配置”并提供前往设置页的入口，不能显示规则生成的伪 AI 文案；
- LLM 尚未生成：显示“今日 AI 分析待生成”；最新完整交易日且已过 `15:15` 时由后台补生成；
- LLM 生成中：百分位和五因子正常显示，解读区显示稳定高度的进行中状态；
- LLM 返回非 JSON、字段缺失或请求失败：最多重试 3 次，仍失败后保存失败时间和原因；
- LLM 失败：显示“今日 AI 分析失败”和重试按钮，不隐藏统计模型结果，也不使用规则文案替代；
- 历史日期没有已生成解读：明确显示“该日未生成 AI 解读”，不临时批量回填。

## 10. 历史验证

提供可复现的离线验证命令，使用与线上完全相同的 Calculator，对每个历史日期执行 walk-forward 计算。验证只评估，不训练权重。

报告按五档情绪区间统计：

- 样本数量；
- 未来 5、10、20 日平均和中位收益；
- 未来 5、10、20 日上涨比例；
- 未来 5、10、20 日最大回撤；
- 冰点和过热状态的平均持续天数。

未来收益只作为历史点之后的验证标签，不进入该历史点的模型输入。报告必须同时输出样本不足提示，不能根据少量极端样本宣称模型有效。

如果各情绪区间没有稳定区分度，线上仍可将该分数用于描述市场温度，但产品文案必须标注“择时有效性未验证”，不得自动调整等权权重掩盖结果。

LLM 输入必须包含该验证结论和样本数量。历史分层无显著区分度时，LLM 不得将冰点或过热描述为已经验证的反向择时信号。

## 11. 测试与验收

### 11.1 后端单元测试

- 五项原始公式和等权综合分；
- 500 日窗口的首个有效点；
- 经验百分位中间秩和并列值；
- 5 日上涨、下跌和零涨跌的方向振幅；
- 500 日价格区间分母为零；
- 缺失或非法 OHLC、成交额；
- 任意未来输入变化不影响该日及之前结果；
- 未收盘当日日 K 过滤；
- 缓存命中、版本失配、过期回退和原子写入；
- 离线验证标签不进入模型计算。
- LLM 输入只包含白名单结构化字段并保留数据日期；
- 输入哈希相同只调用一次，输入变化后允许重新生成；
- 未配置、生成中、成功、Schema 失败、网络失败和三次重试状态；
- `15:15` 前不生成、`15:15` 后只为最新完整交易日生成、容器重启后命中成功缓存；
- LLM 输出不能修改统计分数、等级、权重或交易许可；
- 成功响应按 Pydantic Schema 校验，未知风险姿态和缺失字段被拒绝。

### 11.2 API 和前端测试

- API 默认日期、历史 `as_of`、强制刷新和错误契约；
- 页面独立加载百分位、短线情绪、交易许可和盘中提醒；
- 百分位接口失败不清空其它模块；
- 图表固定显示 `0-100`、冰点区、过热区和最新点；
- 选择历史日期更新五项因子；
- stale 状态显示实际缓存日期；
- 因子名称、分数、原始值和模型说明完整；
- 键盘操作和非颜色状态表达可用。
- AI 解读的未配置、加载、成功、失败和人工重试状态；
- 成功解读展示结论、驱动、背离、历史位置、风险姿态、观察条件和元数据；
- LLM 错误不影响走势图、五因子和现有情绪模块。

## 12. 非目标

- 不训练机器学习、PCA 或预测模型；
- 不使用未来收益调整五项权重；
- 不提供盘中估算百分位；
- 不把新分数混入现有 `emotion_score` 或交易许可；
- 不触发模拟下单、自动下单或仓位调整；
- 不新增独立情绪路由；
- LLM 不修改统计分数、等级、权重或交易许可；
- LLM 不推荐具体股票，不输出仓位比例或自动交易动作；
- 不复制 `stock-sentiment-cn` 的报告脚本、固定评分或通知凭证；
- 不在本期增加社交媒体情绪、北向资金、龙虎榜或新闻因子。
